# 主动算法交易痕迹因子体系：隔日与日内双窗口下的 47 个因子逻辑

## 1. 这套因子到底在识别什么

本项目需要从逐笔委托、成交、撤单和动作发生前的订单簿中，寻找**主动算法交易留下的行为痕迹**。这里的“主动”主要指：策略会根据价格、排队位置、盘口深度、成交进度和市场冲击，持续调整订单，而不是机械地每隔固定时间下一笔。

因子识别的是行为，不是参与者身份。高分可以说明订单流很像精细的程序化执行，但不能仅凭这一项断言它来自量化私募、高频交易账户或某个具体机构。普通投资者、券商自动风控、做市、被动切片和突发消息下的共同反应，都可能形成局部相似的痕迹，所以本项目把证据从简单到复杂分成六层，并另外构造五个对照因子。

截至 2026-07-23，本次总结按当前项目口径覆盖以下 **47 个已构建主因子**：

| 层级 | 因子范围 | 数量 | 主要问题 |
|---|---:|---:|---|
| 第一层：单订单生命周期 | f01—f07 | 7 | 一张订单撤得多快、撤了多少、是否成交 |
| 第二层：撤单—重挂配对 | f10—f21 | 12 | 撤单前后是否出现可解释的替代订单 |
| 第三层：主动成交与流动性获取 | f22—f28 | 7 | 订单是否主动、快速并成体系地获取流动性 |
| 第四层：多步订单链与行为 motif | f29—f41 | 13 | 多次改挂是否组成有经济含义的执行路径 |
| 第五层：盘口事件响应 | f42—f47 | 6 | 动作是否是对前序盘口冲击的合理反应 |
| 第六层：完整执行闭环 | f51—f52 | 2 | 是否形成完整改挂闭环，并最终改善成本 |
| **合计** |  | **47** |  |

本期未纳入 f08、f09、f48、f49、f50 和 f53。编号不连续是为了与最初设计和预注册清单保持一致。

上表的 47 个是**经济逻辑族**，不是最终列数。基础注册表还会按 count/qty/amount、时钟、时序及响应方向等规格展开，并加上五个 control：基础 active inventory 为 201 个 record；Q2 study 排除上述六个 base factor 的 27 个 record 后，正式执行集合固定为 **174 个 study-active record**。这 174 个 record 都会在 `open1h_v1` 和 `open3m_v1` 上各生成一份窗口化因子，因此本次研究不是“只有隔日因子”，而是同一套因子身份对应两条严格分离的时间尺度。

*需要区分“因子逻辑与代码已经构建”和“已经完成真实数据定标并可直接用于生产”。目前这些因子的订单级 scorer、共享生命周期、动作前盘口、配对、链条、冲击和聚合框架已完成相应代码及离线合成验证；部分历史尺度、覆盖率、共同冲击控制和真实全市场分布仍需服务器数据完成定标与验收。*

## 2. 所有因子的统一计算口径

### 2.1 两个因子窗口都必须严格因果

研究范围固定为深交所股票。本次不是一个窗口，而是两个互不混用的正式因子窗口：

| 因子链 | 窗口 ID | 因子证据窗口 | 信息截止时点 |
|---|---|---|---|
| 隔日 | `open1h_v1` | \([09{:}30{:}00,10{:}30{:}00)\) | D1 10:30 |
| 日内 | `open3m_v1` | \([09{:}30{:}00,09{:}33{:}00)\) | D1 09:33 |

两者都在 09:30:00 开始，右端点均排除。09:30 前的消息只能用于恢复开盘订单簿，不能成为因子证据。对 `open1h`，10:30 及以后不能补生命周期；对 `open3m`，09:33 及以后同样不能参与评分、寻找替代单或延长订单链。

例如，09:32:59 的订单在 09:33:01 被撤。虽然事后可以看到这次撤单，但在日内因子的 09:33 信息时点尚不可知，因此它对依赖后续撤单的 `open3m` 因子应按右删失处理；不能因为隔日 `open1h` 版本后来观察到了这次撤单，就反写日内版本。隔日和日内因子是两次独立的因果构建，不是把一小时结果简单裁成前三分钟。

订单簿状态也必须取动作发生前的 \(t^-\) 状态。若用动作后的盘口解释动作本身，就会把结果倒灌进原因。

### 2.2 从原始消息还原订单生命周期

每张窗口内新订单都按交易所因果顺序还原：

- 初始申报量、价格、方向和订单类型；
- 每一笔成交和每一次撤单；
- 每个子事件发生前后的剩余量；
- 首次、末次成交和撤单时点；
- 在本窗口终点 \(T_w\) 是已成交、部分成交后撤、未成交撤单，还是仍然挂着；其中 `open1h` 的 \(T_w=10{:}30\)，`open3m` 的 \(T_w=09{:}33\)；
- 动作前的买一卖一、价差、深度、不平衡、microprice 和前方排队量。

对数据质量正常的订单，数量守恒必须满足：

\[
Q_i^0=Q_i^{tr}+Q_i^{ca}+Q_i^{live}
\]

其中 \(Q_i^0\) 是初始申报量，\(Q_i^{tr}\) 是窗口内累计成交量，\(Q_i^{ca}\) 是窗口内累计撤单量，\(Q_i^{live}\) 是 \(T_w\) 时仍在簿上的数量。引用缺失、重复终结、超额成交或撤单等异常不会被强行修剪成正常值，而是从相关因子的可判定集合中剔除。

### 2.3 不是硬标签，而是 0 到 1 的软证据

每个因子 \(k\) 先对订单 \(i\) 给出一个分数：

\[
S_{i,k}\in[0,1]
\]

- 0 表示没有观察到该类痕迹；
- 1 表示证据非常强；
- 中间值表示证据强弱。

这种设计避免把 0.99 秒和 1.01 秒、9,500 股和 10,000 股粗暴地切成完全不同的两类。常用的连续积木包括：

\[
\operatorname{sat}(x;c)=1-e^{-x/c}
\]

\[
K_t(\Delta)=\exp\left[-\left(\frac{\Delta}{\theta}\right)^p\right]
\]

前者让证据随强度增加但逐渐饱和，后者让时间上越接近的动作联系越强。时间距离既可以是自然秒数，也可以是两个动作间经历的事件数；混合时钟用二者的几何平均，兼顾活跃股和冷门股。

同一订单可能同时参与多个候选配对或多个冲击。在同一个因子内部，默认取最大证据：

\[
S_i=\max_m E_{i,m}
\]

不会把同一张订单重复相加到 1 以上。

### 2.4 每个逻辑同时输出 count、qty 和 amount

对某因子的可判定订单集合 \(\mathcal O_{s,d}^{(k)}\)，统一输出三种股票—日软占比：

\[
F_{s,d,k}^{count}
=
\frac{\sum_i S_{i,k}}{\sum_i 1}
\]

\[
F_{s,d,k}^{qty}
=
\frac{\sum_i Q_i^0S_{i,k}}{\sum_i Q_i^0}
\]

\[
F_{s,d,k}^{amount}
=
\frac{\sum_i V_i^0S_{i,k}}{\sum_i V_i^0},
\qquad
V_i^0=Q_i^0P_i^{ref}
\]

三种口径使用完全相同的订单级分数，只是权重不同：

- count 更接近“疑似行为在订单消息中有多普遍”；
- qty 更接近“有多少申报股数受到这类行为控制”；
- amount 更接近“有多少名义资金规模受到这类行为控制”。

普通限价单的 \(P_i^{ref}\) 使用有效申报价；市价类和特殊订单使用进入前可见的本方或对手方最优价。拿不到可信参考价时，只从 amount 口径剔除，不影响 count 和 qty。

“不可判定”不等于 0。每个口径都有自己的 coverage；分母无效或经济量不可定义时结果为缺失值，而不是“没有算法痕迹”。同时还保存 \(\sum S_i\)、\(\sum Q_i^0S_i\) 和 \(\sum V_i^0S_i\) 三种绝对活动量，以区分“行为占比高”与“行为总规模大”。上述分数、分母、右删失和 coverage 必须在两个窗口内分别计算。

## 3. 隔日与日内两条正式因子链

### 3.1 同一经济逻辑，形成两份独立因子

对任一 study-active record \(r\)，项目分别构造：

\[
F_{s,d,r}^{open1h}
=
\operatorname{Aggregate}
\left(
\{S_{i,r}^{open1h}:i\in\mathcal O_{s,d,r}^{open1h}\}
\right)
\]

\[
F_{s,d,r}^{open3m}
=
\operatorname{Aggregate}
\left(
\{S_{i,r}^{open3m}:i\in\mathcal O_{s,d,r}^{open3m}\}
\right)
\]

二者使用相同的经济公式和 record 身份，但订单集合、生命周期终点、配对候选、订单链、盘口冲击、参数和可判定分母均按各自窗口重新构造。因此，`open3m` 因子表达的是“开盘前三分钟已经显现出的算法痕迹”，`open1h` 因子表达的是“开盘后一小时累计形成的算法痕迹”。

这一区别会实质改变因子值。例如，一张 09:32:50 申报、09:34 撤单并重挂的订单，可以进入 `open1h` 的 f03、f10 或链条因子，却不能利用 09:33 后的动作进入对应 `open3m` 因子。日内版本不会偷借隔日版本更长的观察期。

### 3.2 日内因子：open3m 绑定 09:44—09:45 的一分钟 VWAP

日内正式评价 ID 为：

```text
intraday_open3m_vwap0944_0945_v1
```

它只允许把 `open3m_v1=[09:30:00,09:33:00)` 因子与 `intraday_vwap_0944_0945_from_0933_v1` 标签配对。日内收益首先取严格早于 09:33 的最后一笔有效成交价：

\[
P_{s,d}^{anchor}
=
P\left(
\max\{t<09{:}33{:}00:
P_t>0,\ Q_t>0\}
\right)
\]

再用 09:44:00（含）到 09:45:00（不含）的有效成交计算数量加权 VWAP：

\[
P_{s,d}^{VWAP}
=
\frac{
\sum_{t\in[09{:}44,09{:}45)}P_tQ_t
}{
\sum_{t\in[09{:}44,09{:}45)}Q_t
}
\]

最终唯一的正式日内标签为：

\[
r_{s,d}^{intra}
=
\frac{P_{s,d}^{VWAP}}{P_{s,d}^{anchor}}-1
\]

边界纪律如下：

- 09:33:00 的成交既不进入 `open3m` 因子，也不能成为 anchor；
- 09:44:00 的成交进入目标 VWAP，09:45:00 的成交不进入；
- 缺少严格早于 09:33 的有效成交，或目标一分钟没有正价格、正数量的有效成交，标签为 NaN；
- 09:33 到 09:44 之间的订单流不能进入因子，只能作为因子形成后、目标实现前的市场演化；
- 日内统计的 Newey–West lag 固定为 1。

因此，日内链回答的是：

> 开盘前三分钟已经出现的主动算法痕迹，能否解释从 09:33 可得价格到约 09:45 可实现 VWAP 的短时横截面收益？

### 3.3 隔日因子：open1h 绑定 formal9

隔日正式评价 ID 为：

```text
interday_open1h_formal9_v1
```

它只允许把 `open1h_v1=[09:30:00,10:30:00)` 因子与 `formal9_d1_1030_v1` 标签族配对。以 D1 10:30 价格为信息锚，分别考察 D2、D3、D4 的 open、close 和 VWAP，共九个正式收益标签。三个持有期的 Newey–West lag 分别固定为 1、2、4。

隔日链回答的是“早盘一小时的算法痕迹是否具有未来一至三个交易日的横截面解释力”，与日内链的十几分钟预测问题不是同一时间尺度。

### 3.4 两个窗口分别定标，不能借用一小时参数

定标期统一为 2026-03-23 至 2026-03-27，正式评价期为 2026-04-01 至 2026-06-30，但两个窗口的结构参数和历史统计物理隔离：

- `open1h` 与 `open3m` 分别冻结 theta、饱和尺度、事件活动率、pair 和历史 baseline；
- 两者可以共享仅由定标周 L1 `turnover_rate` 生成的 low/mid/high 流动性组成员表；
- 生命周期、撤单强度、响应、配对、链条和 placebo 统计不能跨窗口共享；
- `open3m` 的日内时间自然只有三个 60 秒桶，不能越过 09:33 为凑样本；
- 定标阶段禁止读取收益、标签、target、future 或 IC，也不能根据 Q2 表现重分箱、改参数或筛因子。

工程上可以对同一交易日的 L2 数据只抽取一次，再让 `open1h`、`open3m` 和标签消费者顺序读取同一份日载荷；这只是原始数据复用，不代表两个窗口可以共享带窗口语义的派生产物。

### 3.5 174 个 record 在两条链上都必须有结果

Q2 执行 inventory 为 174 个 study-active record。每个 record 分别进入两条 evaluation，因此完整评价单位是：

\[
174\ \text{records}\times2\ \text{evaluations}
=348\ \text{record-evaluation bundles}
\]

这不表示又发明了 348 种经济逻辑，而是 174 个冻结因子规格在两个时间窗口上的平行实例。正式流程要求服务器最终执行 174 本 Notebook，每本对应一个 record，并同时包含：

- 隔日九标签 summary、逐日/周度 IC、分组收益、100-bin 和稳定性；
- 日内单标签 summary、逐日 RankIC、周度 IC、分组收益、100-bin 和单调性；
- 两条链各自的有效股票日、缺失率、零值、唯一值和 tie 诊断；
- 对应 window、label、config、registry、revision 和 result manifest 的身份哈希。

正式 evaluator 会报告样本数、coverage、Spearman RankIC、mean RankIC、未年化 ICIR、正 IC 日占比、Newey–West t、Pearson 对照、40 分组和 100 等宽 bins，但不会根据结果自动 retain/reject。

严格禁止把 `open1h` 因子临时配日内标签，或把 `open3m` 因子配 formal9。若要研究交叉组合，必须另建预注册 evaluation，不能污染本次 Q2 正式结果。

## 4. 配对、改价和盘口响应的公共语言

后面很多因子会重复使用以下量：

\[
C_i=\frac{Q_i^{ca}}{Q_i^0},\qquad
E_i=\frac{Q_i^{tr}}{Q_i^0},\qquad
U_i=1-E_i
\]

它们分别是撤单比例、成交比例和未成交比例。

若旧订单 \(i\) 与疑似替代订单 \(j\) 同方向，数量相似度为：

\[
K_q(i,j)=
\exp\left[
-\frac{\left|\log\frac{Q_j^0+\epsilon}{Q_i^{ca,event}+\epsilon}\right|}
{\theta_q}
\right]
\]

这里比较的是这一次真正撤掉的数量，而不是机械地比较两张订单的原始申报量。

令买单方向 \(z_i=1\)，卖单方向 \(z_i=-1\)，则带方向的改价为：

\[
G_{ij}=z_i(P_j-P_i)
\]

- \(G_{ij}>0\)：买单提价或卖单降价，变得更激进；
- \(G_{ij}<0\)：买单降价或卖单提价，变得更节省价格；
- \(G_{ij}=0\)：同价刷新。

基础撤单—重挂连接为：

\[
L_{ij}^{full}
=
\mathbf 1(z_i=z_j)
\cdot K_t^{w_t}
\cdot K_q^{w_q}
\cdot K_p^{w_p}
\]

研究改价方向时，价格变化本身就是证据，不能再被价格相似核惩罚，因此使用：

\[
L_{ij}^{(-p)}
=
\mathbf 1(z_i=z_j)
\cdot K_t^{w_t}
\cdot K_q^{w_q}
\]

两种合法时序都被考虑：

- cancel-first：先撤旧单，再挂替代单；
- replace-first：先挂替代单，再撤旧单，且替代单在重叠期仍有效或已经成交。

当前正式规格主要使用 local-best 匹配；互为最佳和全局一对一求解器也已具备，但若正式发布必须作为独立版本登记。一个配对分数会同时赋给旧订单和替代订单，一条链的分数会赋给链上全部订单，再按订单自身的申报量和金额聚合。

事件响应类统一使用：

\[
\operatorname{Response}_{e,a}
=
\operatorname{ShockStrength}_e
\cdot K_t(e,a)
\cdot \operatorname{ActionConsistency}_{e,a}
\]

它同时要求：冲击足够强、动作发生得足够及时、动作方向在经济上说得通。“快”本身不是充分证据。

## 5. 第一层：单张订单生命周期因子

这一层最容易计算和解释，但误判也最多，适合作为基础刻度。

1. **f01 cancel_presence：是否发生撤单。**

    \[
    S_i=\mathbf 1(Q_i^{ca}>0)
    \]

    它保留最朴素的撤单基线，用来判断复杂因子比“只看有没有撤单”多提供了多少信息。撤单原因很多，所以它不能单独证明主动算法。

2. **f02 cancel_fraction：撤单占原订单的比例。**

    \[
    S_i=C_i
    \]

    1,000 股只撤 10 股与整单撤掉不再等价，比 f01 更能反映订单被放弃或重新部署的程度。

3. **f03 cancel_urgency：撤得是否又快又彻底。**

    \[
    S_i=\mathbf 1(Q_i^{ca}>0)\cdot K_t(t_i^o,t_i^{first\ cancel})\cdot C_i
    \]

    当前可运行主规格使用自然时间和事件时间结合的 hybrid 时钟。它把“撤单速度”和“撤单比例”合并，避免只因有一小笔很快的部分撤单就给高分。

4. **f04 subhuman_latency_cancel：物理上极短延迟的撤单。**

    \[
    S_i=
    \mathbf 1(Q_i^{ca}>0)\cdot C_i\cdot
    \exp\left[-\left(\frac{\Delta t_i^{phys}}{\theta_{phys}}\right)^p\right]
    \]

    该因子只用交易所 `data_time` 的自然时间，不用事件活跃度归一化。Q2 正式 inventory **只纳入预注册的 `f04_50ms_v1` 规格**，即在软衰减核中固定 \(\theta_{phys}=50\text{ms}\)；这不是“50ms 以内算命中、超过 50ms 算不命中”的硬阈值。以本次冻结的 \(p=1\) 为例，撤单延迟为 50ms 时，时间核权重为 \(e^{-1}\approx0.368\)；100ms 时仍有 \(e^{-2}\approx0.135\)，最终再乘撤单比例 \(C_i\)。

    `f04_20ms_v1` 之所以不进入正式 inventory，是因为数据质量预检发现 `data_time` 只落在 10ms 网格上：20ms 仅相当于两个时间戳刻度，过度接近数据分辨率，容易把离散化误差当成物理超低延迟。因此该规格在注册表中保留为 `rejected` 审计记录；50ms 则是本次事前冻结并运行的**结构尺度**，不是五日样本定标出的最优值，也不是交易所或业务上的“接受标准”。f04 只能为自动化链路提供较强行为证据，仍可能来自券商风控或自动撤单机制。

5. **f05 fleeting_unfilled：短命、撤得多、又基本没有成交。**

    \[
    S_i=K_t(t_i^o,t_i^{first\ cancel})\cdot C_i\cdot U_i
    \]

    它描述“挂出观察后很快退出”的订单，比单纯快撤更接近试探或精细执行。

6. **f06 near_touch_fleeting：盘口附近的短命未成交单。**

    \[
    S_i=f05_i\cdot K_{bbo}(i)
    \]

    \(K_{bbo}\) 随订单距本方最优价的档数增加而衰减。只有真正靠近价格竞争和排队位置的 fleeting 订单才会得到高分。

7. **f07 partial_fill_exit：部分成交后迅速撤掉剩余量。**

    \[
    S_i=
    \mathbf 1(0<E_i<1)
    \cdot K_t(t_i^{last\ fill},t_i^{cancel})
    \cdot C_i
    \]

    它捕捉“拿到一部分成交后立即重新决策”的动作，比完全未成交后的快撤更接近对剩余任务的主动管理。

## 6. 第二层：撤单—重挂配对因子

这一层不再只看一张订单，而是判断撤单前后是否出现方向、时间、数量和价格都能解释为替代关系的新订单。

1. **f10 soft_repost：基础软重挂。**

    \[
    S_i=\max_j L_{ij}
    \]

    同时考虑 cancel-first、replace-first 和两者合并版本。它不要求“1 秒内、数量完全相同、价格完全相同”，而让证据随差异连续衰减，是后续多数配对和链条因子的基础。

2. **f11 urgent_repost：原订单本身短命，而且很快被高质量订单替代。**

    \[
    S_{ij}=L_{ij}\cdot K_t(t_i^o,t_i^{cancel})
    \]

    它把快撤与重挂连接起来，比单独的 f03 或 f10 更严格。

3. **f12 residual_size_repost：按剩余任务量重挂。**

    数量核比较新订单 \(Q_j^0\) 与旧订单撤单前的剩余量，而不是比较两张订单的原始申报量。它适合识别“原订单成交一部分后，把尚未完成的部分重新部署”。

4. **f13 same_level_refresh：同价刷新。**

    \[
    S_{ij}=L_{ij}^{(-p)}
    \cdot \exp\left(-\frac{|P_j-P_i|}{tick}\right)
    \]

    价格只计算一次，避免在 \(L^{full}\) 和同价项中重复奖励。同价撤挂会失去原有时间优先级，因此该因子只说明刷新暴露量，不自动等于队列优化。

5. **f14 price_saving_repost：向更省钱方向退价。**

    \[
    S_{ij}=L_{ij}^{(-p)}
    \cdot \operatorname{sat}((-G_{ij})_+;c_p)
    \]

    买单降价或卖单提价时得分，表示策略愿意牺牲一部分成交概率以争取更好的成交价格。

6. **f15 aggressive_repost：向更容易成交的方向追价。**

    \[
    S_{ij}=L_{ij}^{(-p)}
    \cdot \operatorname{sat}((G_{ij})_+;c_p)
    \]

    买单提价或卖单降价时得分，反映策略为提高完成概率而增加价格让步。

7. **f16 queue_improving_repost：重挂后前方排队量是否减少。**

    \[
    S_{ij}=L_{ij}^{(-p)}
    \cdot
    \operatorname{sat}\left(
    \frac{(A_i^{old}-A_j^{new})_+}{A_i^{old}};c_A
    \right)
    \]

    \(A_i^{old}\) 是旧订单撤前的前方排队量，\(A_j^{new}\) 是新订单进入前的前方排队量。若旧订单原本就在队首，分母为 0 时改善项定义为 0，不用任意小数伪造改善。

8. **f17 split_reallocation：一撤多挂的拆分再分配。**

    对旧订单附近的一组替代订单 \(J\)，比较：

    \[
    \sum_{j\in J}Q_j^0
    \quad\text{与}\quad
    Q_i^{ca,event}
    \]

    同时看组内时间紧密度、总量守恒和价格层次。它能识别把一笔剩余任务拆成多个价位或多个子单的行为。

9. **f18 merge_reallocation：多撤一挂的合并再分配。**

    它与 f17 对称：比较一组相近撤单的总撤量与一张新订单的申报量，识别算法收回多个小挂单后集中部署。

10. **f19 cancel_to_marketable：撤掉被动单后切换为主动成交。**

    \[
    S_{ij}=L_{ij}^{(-p)}\cdot M_j\cdot X_j
    \]

    该因子只定义 cancel-first。\(M_j\) 衡量新订单进入前是否可成交，\(X_j\) 衡量它随后成交得多快、成交量有多大。它比“新订单是 taker”多了一步可解释的执行方式切换。

11. **f20 fill_then_reallocate：部分成交—撤余量—重新部署。**

    \[
    S=
    \left[
    E_i\cdot
    K_t(t_i^{last\ fill},t_i^{cancel})
    \cdot L_{i,J}
    \right]^{1/3}
    \]

    三个环节缺一都会明显降分，因此是高精度、低召回的主动执行闭环。

12. **f21 cancel_repost_burst：撤单簇与新单簇的总量守恒。**

    \[
    K_q^{burst}=
    \exp\left[
    -\frac{
    \left|\log\frac{\sum Q^{new}+\epsilon}
    {\sum Q^{cancel}+\epsilon}\right|}
    {\theta_q}
    \right]
    \]

    它不强求逐张配对，而是检查局部窗口内同方向的一批撤单与一批新单是否在总量、时间和价格路径上构成整体重新布置。

## 7. 第三层：主动成交与流动性获取因子

令 \(M_i\in[0,1]\) 表示订单进入前的可成交性，令：

\[
X_i=
\frac{1}{Q_i^0}
\sum_{r\in trades(i)}
Q_rK_t(t_i^o,t_r)
\]

表示按时间衰减加权的快速成交比例。它不限制一张订单最多产生几条成交记录。

1. **f22 marketable_fill：进入时可成交，并且迅速成交。**

    \[
    S_i=M_i\cdot\min(1,X_i)
    \]

    它是“主动获取流动性”的基础因子，但普通投资者的市价买卖也会命中，因此不是高纯度算法身份因子。

2. **f23 instant_fill_fraction：成交比例与成交速度的联合分数。**

    \[
    S_i=X_i
    \]

    它不强制要求进入前已可成交，可覆盖隐藏流动性、盘口同步变化或数据细节导致的快速成交。

3. **f24 depth_consuming：快速成交量相对对手方可见深度有多大。**

    \[
    S_i=
    M_i\cdot
    \operatorname{sat}\left(
    \frac{Q_i^{fastfill}}{D_i^{opp}+\epsilon};c_D
    \right)
    \]

    同样的 10 万股在厚盘口和薄盘口中的意义不同，因此必须用动作前深度归一化。

4. **f25 multi_level_sweep：是否快速扫过多个价格层。**

    \[
    S_i=
    M_i\cdot\min(1,X_i)
    \cdot\operatorname{sat}(L_i^{price}-1;c_L)
    \]

    触及层数的影响连续饱和，不使用“至少三档”之类硬门槛。

5. **f26 aggressive_burst：同方向主动成交簇。**

    把正式判定为主动方的同方向订单按软时间关系聚成簇，综合时间紧密度、子单量稳定性、累计主动成交量相对对手深度和簇规模。它识别的是一段成体系的主动流动性获取，而不是单笔大单。

6. **f27 child_execution_sequence：同方向子单执行序列。**

    进一步检查方向是否稳定、数量是否呈可解释切片、任务是否逐步完成，以及间隔是否与市场成交量或流动性状态相关。它可以说明程序化执行，但主动算法和被动执行算法都可能出现，必须与响应、改挂或成本因子联合解释。

7. **f28 liquidity_switching_cycle：在 maker 与 taker 之间切换。**

    寻找“先被动挂单等待—未完成后撤单—改为更激进订单—必要时再回到被动挂单”的链条，以被动起点、主动程度上升、完成度、链边质量和链长共同评分。

## 8. 第四层：多步订单链与行为 motif

把订单视为节点，把高质量撤单—重挂关系视为带权边，就可以构成订单链。链条因子不要求每一步完全同量、同价或固定在一秒内，而是判断整条路径是否连续、守恒且具有经济含义。

1. **f29 soft_run_membership：订单在软连接图中的参与度。**

    \[
    S_i=
    1-\exp\left[
    -\frac{\sum_jL_{ij}+\sum_hL_{hi}}{\theta_R}
    \right]
    \]

    一条高质量相邻边就可得到中等分数，关系越多、质量越高则逐渐饱和。

2. **f30 long_run_saturation：长链质量。**

    \[
    R_C=
    \overline L_C
    \left[
    1-\exp\left(-\frac{m_C-1}{\theta_m}\right)
    \right]
    \]

    两节点链不会被硬判为 0，长链也不会无限加分；平均边质量防止很长但很松散的随机链得高分。

3. **f31 high_confidence_run：由薄弱环节约束的高置信链。**

    当前版本使用链内边分数的 20% 分位数乘链长饱和分，而不是只看平均值。少数高分边不能掩盖大量弱连接。

4. **f32 repricing_run：连续改价链。**

    检查链上的价格变化是否沿同一经济方向持续，例如逐步追价、逐步退价或随市场移动，而不是来回随机跳动。得分由边质量、非零改价比例、方向一致性和改价尺度共同决定。

5. **f33 bbo_following_run：跟随最优报价维护目标位置。**

    对每张订单计算其价格相对本方 BBO 的 tick 距离。如果 BBO 变化时订单也改价，但相对 BBO 的目标距离保持稳定，则说明策略可能在动态维护报价位置。

6. **f34 queue_optimizing_run：连续多步改善队列位置。**

    对链上每条边读取旧订单撤前和新订单进入前的 FIFO 前方量，综合多步队列改善、链边低分位、数量守恒和链长。它是精细管理成交概率的直接证据，但高度依赖订单簿重建质量。

7. **f35 cost_saving_run：连续向更省钱方向改价。**

    要求买单多次降价或卖单多次提价，同时任务量大致守恒并最终有成交。它试图区分“为了成本耐心改挂”与单纯改变方向判断。

8. **f36 aggressive_escalation_run：逐步提高成交紧迫度。**

    若链上订单的可成交性 \(M_i\) 或带方向价格让步持续上升，说明策略可能先尝试被动成交，在时间或市场条件收紧后逐步提高价格让步。

9. **f37 execution_completing_run：任务量是否在链中逐步完成。**

    以根节点申报量作为任务量代理，观察链内累计成交相对任务量的完成度，并同时要求数量守恒、边质量和链长。它把“频繁发消息”与“围绕一个目标执行”区分开。

10. **f38 state_adaptive_run：订单链是否随市场状态调整。**

    检查每次改挂是否紧随 BBO、价差、排队量、大额成交、不平衡或短时波动变化。相同订单链若没有可解释冲击，得分为 0；只有链质量和状态响应同时存在才给分。

11. **f39 two_sided_quote_management：双边报价维护。**

    要求买卖两侧都存在真实匹配边、旧单和新单都靠近 BBO、两侧动作近同步、调整幅度对称，并且由不同的可解释冲击触发。该 scorer 已实现并通过合成拒绝测试，但正式记录仍保持 experimental/blocked：数据没有账户 ID，无法证明买卖两侧属于同一参与者，也尚不能在单股票阶段充分排除共同市场冲击。因此它只能称为“股票层面的实验 motif”，不能直接称为同一做市商的双边管理。

12. **f40 iceberg_like_replenishment：成交后在同价或近价快速补量。**

    \[
    S\sim K_t\cdot K_p\cdot K_q\cdot\operatorname{RunLengthScore}
    \]

    它可能对应冰山单、隐藏母单或程序化补单，但也可能是被动执行，因此保留为程序化交易诊断，不自动解释为主动算法。

13. **f41 reactive_nonperiodic_run：事件驱动而非钟表驱动。**

    \[
    R_C=
    \operatorname{StateResponse}_C
    \cdot
    \left(1-\operatorname{ClockPeriodicity}_C\right)
    \]

    实现中非周期部分可由申报间隔的 \(CV/(1+CV)\) 表示。不规则本身没有意义，必须与盘口冲击响应相乘；否则人工随机交易也会得高分。

## 9. 第五层：盘口事件响应因子

这一层不只问“订单做了什么”，而是问“它是否在某个已发生的冲击之后，及时做出了方向合理的调整”。

1. **f42 best_quote_response：最优报价变化响应。**

    分为本方 BBO 和对手方 BBO 两个版本。当最优价变化后，观察订单是否撤掉失去竞争力的报价、跟随新最优价重挂、避免变得过于激进，或在新的目标价格层重新部署。

2. **f43 queue_dilution_response：排队被稀释后的调整。**

    \[
    S_i=
    \operatorname{ShockStrength}_{queue}
    \cdot K_t
    \cdot\operatorname{ActionConsistency}_{queue}
    \]

    简版冲击强度看订单前方量相对撤单时前方量的净增幅，动作一致性则看重挂是否把订单移到更有利的队列位置；formal v2 进一步复用统一的 `market_impact` 冲击定义。它比“看到撤单重挂”更直接地说明策略在管理排队细节。

3. **f44 depletion_or_large_trade_response：深度消耗或大成交后的响应。**

    冲击强度取“可见对手方深度被消耗的比例”和“本笔成交量相对严格前序正常成交量的异常程度”中的较强者，再观察订单是顺势追击，还是撤退以避免不利成交。固定用“超过多少股”无法跨股票比较。

4. **f45 spread_regime_response：价差状态变化响应。**

    收窄和扩大是两个独立版本：

    - 价差收窄时，奖励变得更激进或转向 taker 的动作；
    - 价差扩大时，奖励撤退或转向 maker 的动作。

    同一个动作在两个版本中的经济含义相反，因此不能混成一列。

5. **f46 imbalance_flip_response：盘口不平衡翻转响应。**

    只有不平衡指标严格过零才算翻转，冲击强度使用：

    \[
    \frac{|I_{after}-I_{before}|}{2}
    \]

    再乘以与新方向一致的撤退、追价或重新部署证据。轻微过零不会自动得到满分。

6. **f47 volatility_jump_response：短时波动跳升后的风险调整。**

    用短窗 microprice RMS 变化相对严格前序正常窗口的超额比例衡量冲击，只奖励缩量、退价等降低暴露的动作。当前是 raw 响应版，尚未声称已经消除全市场新闻或指数冲击；消息日中所有理性参与者同步调整仍是主要误判来源。

## 10. 第六层：完整执行闭环

这一层追求高精度，不满足于某一个局部动作，而要求多环证据共同成立。

1. **f51 active_revision_cycle：完整主动改挂闭环。**

    \[
    S=
    \left(
    K_{bbo}
    \cdot\operatorname{CancelEvidence}
    \cdot L_{repost}
    \cdot\operatorname{CostOrQueueLogic}
    \cdot\operatorname{ResponseOrNonrandomness}
    \right)^{1/5}
    \]

    五个环节分别要求：旧订单靠近盘口、撤单证据可信、存在高质量替代、改价或队列变化有经济含义、动作能够由前序盘口事件解释。几何平均使缺失任何关键环节都会显著降分，但不要求每一项都等于 1。

    实现上，“响应面完整但没有符合条件的响应”记为 0；响应面、BBO、队列或生命周期本身不可观测则保留为缺失，二者不能混淆。

2. **f52 cost_improving_completed_cycle：最终实现成本改善的执行闭环。**

    对撤单—重挂—最终成交链，把实际成交价与撤单时旧价、动作前 mid、microprice、因果短窗 VWAP/TWAP 和当时可见的立即成交成本比较。买单实际成交必须更低、卖单必须更高，而且要优于全部已登记基准，才产生正向成本改善。

    \[
    S_C=
    \operatorname{RevisionQuality}_C
    \cdot\operatorname{Completion}_C
    \cdot
    \left[
    1-\exp\left(
    -\frac{(\operatorname{signed\ cost\ improvement})_+}
    {\operatorname{spread\ or\ volatility\ scale}+\epsilon}
    \right)
    \right]
    \]

    该因子最接近“算法反复优化几个基点”的直觉，但它只识别**成功的主动执行**。失败的主动算法也可能真实存在，所以 f52 是高纯度旁证，不能替代前面的行为因子。

## 11. 五个配套对照因子

以下 control 不计入上述 47 个主因子。它们的作用不是识别主动算法，而是检验主因子是否其实只捕捉了普通程序化切片、一次性方向交易或随机撤单。

1. **ctrl01 periodic_slicing：固定周期切片。**

    \[
    S_C=
    \frac{1}{1+CV(\Delta t_C)}
    \cdot(1-\operatorname{StateResponse}_C)
    \]

    间隔越稳定、与盘口事件关系越弱，越像 TWAP 一类被动执行。

2. **ctrl02 fixed_clip：固定子单量。**

    用链内子单量的对数标准差衡量数量稳定性，并由状态响应和重挂证据连续折减。高分说明订单更像固定 clip 切片。

3. **ctrl03 volume_participation：跟随市场成交量。**

    比较动作前近期市场成交量与后续子单量或频率的对数相似度，并结合簇规模。它更像 POV 或 VWAP 类算法；缺少历史 activity baseline 时会 fail closed，不会用 0 冒充结果。

4. **ctrl04 one_shot_taker：一次性主动成交。**

    要求订单进入前可成交、快速完成、没有撤单，且宽窗口内没有同方向执行序列。它帮助区分“简单方向性大单”和“会持续管理订单的主动算法”。

5. **ctrl05 random_cancel：孤立或缺乏替代证据的撤单。**

    配对正式版的核心为：

    \[
    S_i=C_i\left(1-\max_jL_{ij}^{full}\right)
    \]

    只有撤单比例高、又找不到可信替代订单时才高分。它只给撤单订单打分，不把“随机”证据赋给新订单。

对照因子的正确用法是条件比较。例如：

- f10 soft_repost 高、ctrl01 periodic_slicing 也高，可能只是被动切片算法；
- f51 active_revision_cycle 高、ctrl01 低，更接近状态驱动的主动执行；
- f22 marketable_fill 高、ctrl04 one_shot_taker 也高，可能只是一次性方向性交易；
- f01/f02 撤单因子高、ctrl05 random_cancel 也高，说明撤单类证据的误报空间较大。

## 12. 如何整体理解这 47 个因子

这 47 个因子不是 47 次对同一件事的重复计数，而是从不同证据层回答逐渐更严格的问题：

| 证据层 | 高分说明什么 | 仍不能说明什么 |
|---|---|---|
| 单订单生命周期 | 有快撤、深撤、部分成交后退出等局部行为 | 是否存在持续任务或合理替代 |
| 撤单—重挂配对 | 旧单与新单在时序、数量和价格上可以互相解释 | 连接是否只是高消息率下的巧合 |
| 主动成交 | 有快速成交、吃深度、扫多档和子单执行 | 是否为了精细管理成本 |
| 多步订单链 | 多次动作组成守恒且有方向的路径 | 是否来自同一账户；是否受共同冲击 |
| 盘口响应 | 动作与前序市场冲击在速度和方向上吻合 | 是否优于历史正常反应率 |
| 完整闭环 | 改挂、队列或成本逻辑、响应和完成度同时成立 | 是否覆盖所有主动算法；失败策略可能被漏掉 |

层级越高，通常解释力越强、误报越少，但覆盖率也越低，对数据完整性和参数定标的要求越高。
