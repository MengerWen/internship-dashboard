# 寻找算法痕迹：六篇论文 42 个指标的构建、计算与评估

## 1 项目总况

### 1.1 这一步总目标

只用交易所公开的逐笔委托/成交/撤单数据，度量每只股票每天的**算法交易活动强度**。
方法来自六篇论文（EE、KX、CCP、KKST、HJM、GB，见文末论文清单），它们各自提出了不同的度量思路。
本阶段把六篇论文的方法全部复现（并适当改造）成代码，为求回答两个问题：

1. 这些指标各自在度量什么？彼此之间重复度有多高？
2. 在没有真标签的情况下，有哪些证据能说明它们确实抓到了"机器行为"？

数据流是：原始 order/trade/cancel $\to$ 标准事件 $\to$ 赋予生命周期的订单 $\to$ 42 个股票-日指标 $\to$ 指标：评估/记分 $\to$ PCA 融合。产出五个缓存文件：

## 2 读取 & 标准化 L2 事件 & 打标签

1. 统一事件
   - 把订单、成交、撤单三类消息统一成标准字段：`etype`（事件类型）、`ts`（时间戳）、`code`、`side`（买卖方向）、`price_tick`（价格）、`qty`（数量）、`order_id`，成交事件额外带 `bid_id`/`ask_id`（买卖双方的订单 ID——这是后面判定"谁是主动方"的关键），以及 `event_pos`（当日事件序号，用于毫秒内排序）。
2. 重建生命周期
   - 把同一个 `order_id` 的所有事件串成一份"订单档案"：什么时候提交（`submit_ts`）、报价离盘口几档（`bbo_diff_ticks`）、挂在第几档（`depth_rank`）、报了多少量（`submit_qty`）、量相对当日中位数是大是小（`qty_scaled`）、成交了几成（`fill_ratio`）、是否主动吃单（`is_taker`）、活了多久（`life_s`）、最终结局（`end_state`：成交完 / 撤单 / 部分成交后撤 / 挂到收盘过期）。
   - 口径修正：挂到连续竞价结束的订单按 14:57 计算寿命，不记为 0。
3. 打标签，弱监督思路
   - 档案上生成三类候选标记（注意都是"候选行为"，不是真标签）：EE 快撤/关联标记（`flag_ee_step1/step2/ee`）、CCP 策略性重挂链标记（`flag_ccp_sruns`）、GB 种子标签（`gb_seed_label`：1 正种子 / 0 负种子 / -1 灰区）。

## 3 构建指标：42 个指标与它们的 9 个构造逻辑类（重点）

### 3.1 为什么要先聚类

六篇论文合计产出 42 个指标。但论文之间互相独立，很多指标的构造思路甚至构造公式高度重叠。按照**构造逻辑**归类后，42 个指标归为 9 类；归根结底，真正独立的信息轴只有约 7 条：订单存活时间、订单链式关联、消息强度、盘口扰动、事件响应速度、主动成交/价格冲击、成交成本后果。

以下逐类说明构造方法。记每笔订单 $i$ 的寿命为 $life_i$（提交到撤单/成交完/收盘终止的秒数），当日订单总数为 $N$，订单报量为 $q_i$。

#### 类 1｜短生命 / 快速撤单（11 个指标，重复度最高）

所有成员的底层事件完全相同——**"订单提交后很短时间内被撤"**：

- `ee__step1_share` = $\frac{1\text{ 秒内被撤的订单数}}{N}$。EE 两步法的第一步。
- `kx__short_cancel_lt_1s` / `lt_10s`：同构，阈值分别取 1 秒、10 秒。1 秒版与 `ee__step1_share` 只差寿命口径细节，**几乎是同一个数**。
- `ccp__fleet`：同一事件不做归一化的**日内计数** $\approx$ `ee__step1_share` $\times N$，因此强烈受股票活跃度影响。
- `kx__life_p10/p25/p50/p75/p90`：寿命分布的五个分位数。本样本中 **p10、p25 恒为 0**（大量订单瞬间成交），这两个指标已经退化了、约等于无效。
- `ee__hft_ratio` = $\frac{\lvert S_1 \cup S_2\rvert}{N}$，其中 $S_1$ 是快撤集合、$S_2$ 是关联扩散集合（见类 2）。
- `ee__flag_qty_share`：同一并集换成**报量加权**，$\frac{\sum_{i\in S_1\cup S_2} q_i}{\sum_i q_i}$。

本类真正的自由度约 2 个："寿命左尾有多重" + "快撤单占多少报单量"，其余是阈值/归一化变体。

#### 类 2｜链式重复下单 / 订单簇

核心逻辑：**相似订单 $+$ 时间邻近 $\to$ 连成链**，捕捉"同一算法反复报撤"。

- EE step2：以每个快撤单为根基，在其前后 1 秒内寻找**同代码、同方向、同数量**（默认还要求同通道）的其他订单，全部并入 S2——"抓到一个快手，顺藤摸出同伙"。
- `ccp__sruns`：把"撤单 $\to$ 毫秒级间隔内重新挂单"连成链，链上订单数达到阈值记一条 run；指标 $=$ 当日 run 条数。
- `gb__seed_pos` = $\lvert \text{flag\_ee\_step1} \cup \text{flag\_ccp\_sruns}\rvert$ 的计数。类 1 与类 2 的机械并集，是 GB 训练的簿记量，**不是独立因子**。

#### 类 3｜消息强度 / 订单成交比（OTR）

核心逻辑：**单位成交承载了多少消息**。

- `kx__otr_day` = $\frac{\text{报单数}+\text{撤单数}}{\text{成交笔数}}$
- `ccp__mess`：距同侧最优报价 k 档以内的报单+撤单**计数**；`ccp__can`：其中撤单的计数。can ⊂ mess，且都是未归一化的原始计数，横截面上被股票各自的活跃度严重影响。
- `hjm__msg_per_volume` = $\frac{\text{全部事件数}}{\text{成交股数}}$。OTR 的近亲，分母从笔数换成股数。

#### 类 4｜盘口报价扰动 / 簇发

四个指标全部来自同一条 L1（买一/卖一）时间序列。令 $x_t=\mathbf{1}\{\text{第 }t\text{ 秒盘口价或量有变动}\}$：

- `ccp__quoteint` = $\sum_t x_t$，盘口变动的总秒数。
- `kx__burst_count` / `kx__burst_seconds`：对同一计数做滚动窗口，窗口内变动次数超过阈值记为"簇发状态"；`count` $=$ 进入簇发的次数，`seconds` $=$ 处于簇发的累计秒数。这是 quote stuffing（报价轰炸）视角。
- `ccp__flick`：桶内 1 秒中间价的标准差——前三个量衡量"变动多不多"，这个量衡量"晃动大不大"。

#### 类 5｜事件响应速度

共同模板：**取一个盘口事件 $\to$ 开一个短窗口 $\to$ 统计窗口内的跟随行为**。

- `ccp__sresp`：每次报价改善（买一上抬或卖一下压）后，找窗口内第一个响应事件（同侧追单 / 同侧撤单 / 对改善价的成交）计 1，全日累计——市场对新报价的整体反应速度。
- `kkst__newlevel_maker_lift`：盘口出现不利变动后，窗口内新报单中被标记订单的占比 ÷ 全日被标记占比。
  - $>1$ 说明被标记群体在盘口变化后**抢先**重新报价

#### 类 6｜主动成交 / 知情性

核心逻辑：**taker 行为与价格变动的先后关系**。

- `ccp__ioc`：主动吃单订单计数。
- `kkst__aggressive_ratio_flagged`：被标记订单的主动成交量 / 全部主动成交量。
- `kkst__snipe_taker_lift`：每次不利价格变动**之前**取最近 k 笔成交，算可疑 taker 的量占比，除以全日基准后取平均。
  - $>1$ = 价格要动之前，可疑资金先动手（狙击旧报价）。
- `kkst__flow_price_beta_lag0~5`：令 $f_t$ 为第 $t$ 秒被标记订单的带符号成交量（买 $+$、卖 $-$），$\beta_\ell=\frac{\operatorname{Cov}(f_t,\Delta mid_{t+\ell})}{\operatorname{Var}(f_t)}$——被标记资金流对未来 $0\sim 5$ 秒中间价变动的预测系数。
- `kkst__flow_price_beta0`：**代码级复制** $=$ beta_lag0，纯冗余。

#### 类 7｜成交经济后果（HJM 验证器）

对每笔成交（价格 $p$，成交时中间价 $m_0$，$\Delta$ 秒后中间价 $m_1$，方向 $d=+1$ 买方主动 / $-1$ 卖方主动）：

- `hjm__espread_bps`：有效价差 $ES=d\cdot(p-m_0)/m_0$——成交那一刻付出的即时成本。
- `hjm__rspread_bps`：已实现价差 $RS=d\cdot(p-m_1)/m_0$——扣掉后续价格漂移后，做市方真正赚到的部分。
- `hjm__advsel_bps`：逆向选择成本 $AS=ES-RS$。**恒等式差值**，三者只有两个自由度。
- `hjm__trade_size`：平均单笔成交股数——算法拆单越狠，这个数越小。

这一类与其他类性质不同：它们度量的是市场质量**后果**，不直接刻画算法行为，在本项目中用作验证器。

#### 类 8｜撤单–成交交互（fade / spoof 形态）

每笔成交后开一个短窗口，统计 maker 同侧、同价（或价格带内）的撤单总量 C：

- `kx__fade_partial_prob` = $Pr(C>0)$——成交后同侧同价"缩手"的概率。
- `kx__fade_full_prob` = $Pr(C\ge \text{成交量})$——撤掉的比成交的还多。嵌套关系，恒有 $partial\ge full$。

与类 1 的区别：类 1 只看订单自身活多久，这里看撤单与**成交事件的时间排布**，是本项目唯一直接刻画 spoofing 形态的指标，逻辑独立性较高。

#### 类 9｜弱监督模型综合（GB）

用 11 个订单特征（寿命、离盘口档数、相对报量、成交比例、是否主动、终态、时段等）训练梯度提升模型。正种子 $=$ EE step1 $\cup$ SRuns 命中订单；负种子 $=$ 长寿 $+$ 正常完结 $+$ 不超过中位数量的"类人"订单；其余为灰区。模型输出每笔订单的"疑似算法概率" $\hat p_i$：

- `gb__sum_p` = $\sum_i \hat p_i$——全日"算法嫌疑总量"（按笔）；`gb__sum_p_amount` = $\sum_i \hat p_i \times \text{订单金额}_i$——按金额加权。
- 由于训练标签来自类 1+2、特征与类 1/2/6 重合，GB 分数在构造上就是那些类的**非线性重组**，不是独立信号源。

## 4 指标的最终计算结果（重点）

### 4.1 原始值的量纲问题

先看原始面板（完整 $20\text{ 行}\times 42\text{ 列}$的示例表在展示版）。挑 000001 / 2024-01-02 一行感受量级差异：

| 指标 | 原始值 | 类型 |
| --- | ---: | --- |
| ee__step1_share | 0.023 | 比例 |
| kx__otr_day | 1.89 | 比值 |
| ccp__fleet | 1,984 | 计数 |
| ccp__mess | 96,174 | 计数 |
| ccp__sresp | 26,125 | 计数 |
| hjm__advsel_bps | 2.21 | bps |
| gb__sum_p | 41,914 | 概率和 |
| gb__sum_p_amount | $1.95e+09$ | 金额加权 |

比例型 $\sim 0.02$、计数型 $\sim 10^5$、金额型 $\sim 10^9$：**原始值之间没有任何可比性**。所以所有跨指标比较都先做横截面标准化：对每个指标列，在全部 150 个股票-日样本上计算 z-score（减均值除标准差，$\pm 2.5$ 截断）。标准化后比较的不再是绝对水平，而是"每只股票在该指标上的横截面相对位置"。

1. **计数型指标族在高流动性层整体偏红**（这是废话）；
2. 值得注意，**同一层内部的例外**：某只中/低流动性股票在某族指标上异常突出；
3. **比例型指标（EE 系列、short_cancel）的红蓝分布与计数型明显不同**——"算法活动的总量"和"算法活动的占比"含义不同

#### 规模中和之后还剩什么

### 5.3 GB 模型质量（简述）

GB 模型在训练日和留出日复现种子标签的 $PPV=1.0$、$NPV\approx 1.0$——但这只说明模型学会了种子规则（种子本来就是 EE/SRuns 规则生成的），不代表识别真算法的能力。蒸馏出的决策树第一刀就是"寿命是否超过 1 秒"，其后依次看终态、相对报量、挂单档位——模型自己学到的判别逻辑与手工聚类的类 1/类 2 完全一致，算一个侧面印证。

![GB 分数校准曲线（灰区订单）](assets/algo-footprint-2026-07-07/cell_0020_output_0008.png)

对模型没见过标签的 770 万笔灰区订单按分数分箱检验：分数高的箱子总体上确实更常撤单、活得更短，但**校准曲线不单调**——分数挤在 0.693 附近的极窄区间，个别高分箱回落。结论：GB 分数当**排序**用可以（`gb__sum_p_amount` 也确实是全场稳定性最高的指标），当**概率**解读不行。

### 5.4 稳健性与一致性检验

三个检验共同回答一个问题：**指标量到的是股票的持续属性，还是一日噪声？**

![滞后稳定性曲线](assets/algo-footprint-2026-07-07/cell_0023_output_0002.png)

**（a）滞后稳定性**：今天的股票排名与 $1\sim 4$ 天后排名的 Spearman 相关。gb__sum_p_amount（$0.97\to 0.95$）、ccp__sresp（$0.95\to 0.92$）、kx__fade_partial_prob（$0.91\to 0.91$）几乎不衰减——量的是长期属性；ee__step1_share 从 0.76 衰减到 0.60——有持续成分但混噪声；ccp__sruns（$0.22\sim 0.58$ 波动，事件太稀疏）和 hjm__advsel_bps（$0.11\sim 0.30$，本来就该逐日波动）最弱。

![2024 vs 2025 股票排名一致性](assets/algo-footprint-2026-07-07/cell_0024_output_0002.png)

### 5.7 证据链小结

物理层（被标记订单 40ms 中位延迟、61.6% 快于人类反应下限 vs 未标记组 0%）+ 记分卡（`ee__step1_share` 控制一切计数指标后仍有独立信息）+ 稳健性（核心指标跨日、跨半日、部分跨年稳定）——三者互相独立又互相咬合。在没有真标签的前提下，"**稳定 + 独特 + 符合先验 + 速度证据**"就是当前能给出的最强论证。

方法间的订单级对照显示 GBTOP 独占率最高（97.9%），而 $EE1-EE2$ 仍存在可见印证（Jaccard 0.124）——多方法不是重复计数，而是互补关系。
