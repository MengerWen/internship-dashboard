订单寿命分布

对每个股票-日期算指标

六篇论文共形成 42 个指标，进入记分卡的是 25 个 ee/kx/ccp/gb 方法指标。跨论文前缀看，真正独立的信息轴约 7 条：订单存活时间、订单链式关联、消息强度、盘口扰动、事件响应速度、主动成交/价格冲击、成交成本后果。

9 类构建逻辑卡片

类 1｜短生命 / 快速撤单

核心：订单寿命 $\ell_i$ 的左尾

记每笔订单 $i$ 的寿命 $\ell_i$（提交到撤单/成交完/收盘终止的秒数），$N$ 为当日订单总数，$q_i$ 为报单量。

  • ee__step1_share：提交后 1 秒内被撤的订单占比。$\dfrac{1}{N}\sum_i \mathbf{1}\{\text{被撤}_i \wedge \ell_i < 1\text{s}\}$
  • kx__short_cancel_lt_1s / lt_10s：与上式同构，阈值分别取 $\tau = 1$ 秒和 $10$ 秒。备注：1 秒版与 ee__step1_share 几乎是同一个数，只差寿命口径细节。
  • ccp__fleet：同一事件不做归一化的日内计数 $\sum_i \mathbf{1}\{\text{被撤}_i \wedge \ell_i < 1\text{s}\}$。备注：≈ ee__step1_share × 当日订单总数，因此强烈受股票活跃度影响。
  • kx__life_p10/p25/p50/p75/p90：寿命分布的五个分位数 $Q_{10\%}(\ell), \dots, Q_{90\%}(\ell)$。备注："阈值处的占比"和"分位数"是同一分布的两种读法；本样本中 p10、p25 恒为 0（大量订单瞬间成交），已退化。
  • ee__hft_ratio：EE 两步标记的并集占比 $|S_1 \cup S_2| / N$，其中 $S_1$ 为快撤集合、$S_2$ 为关联扩散集合（见类 2）。
  • ee__flag_qty_share：同一并集换成量加权 $\sum_{i \in S_1 \cup S_2} q_i \big/ \sum_i q_i$。

本类真正的自由度约 2 个："寿命左尾多重"＋"快撤单占多少报单量"，其余是阈值/归一化变体。

类 2｜链式重复下单 / 订单簇

核心：相似订单 + 时间邻近 → 连成链

  • EE step2：以每个快撤单 $i \in S_1$ 为种子，在 $[t_i - w,\ t_i + w]$（$w = 1$ 秒）内寻找同代码、同方向、同数量（默认还要求同通道）的其他订单，全部并入 $S_2$——"抓到一个快手，顺藤摸出同伙"。
  • ccp__sruns：把"撤单 → 毫秒级间隔内重新挂单"连成链，链上订单数 $\ge$ 阈值（run_min_len）记一条 run；指标 = 当日 run 条数——同一算法反复报撤的直接证据。
  • gb__seed_pos：$|\{i : \text{flag\_ee\_step1}_i \vee \text{flag\_ccp\_sruns}_i\}|$。备注：类 1 与类 2 的机械并集，是 GB 训练的簿记量，不是独立因子。

类 3｜消息强度 / 订单成交比（OTR）

核心：单位成交承载的消息量

  • kx__otr_day：$\dfrac{\#\text{报单} + \#\text{撤单}}{\max(\#\text{成交}, 1)}$——每撮合成一笔交易，市场要"喊"多少嗓子。
  • ccp__mess：距同侧最优报价 $\le k$ 档（配置 near_bbo_ticks）的报单+撤单计数；ccp__can：其中撤单的计数。备注：can ⊂ mess，机械相关；两者都未归一化，横截面上被活跃度主导。
  • hjm__msg_per_volume：$\dfrac{\#\text{全部事件}}{\sum_{\text{成交}} q}$。备注：OTR 的近亲，分母从笔数换成股数。

类 4｜盘口报价扰动 / 簇发

核心：L1 序列的变动频率与幅度

令 $x_t = \mathbf{1}\{\text{第 } t \text{ 秒买一/卖一价或量发生变动}\}$。

  • ccp__quoteint：$\sum_t x_t$——盘口变动的总秒数。
  • kx__burst_count / kx__burst_seconds：对同一计数做 $w$ 秒滚动和，$\sum_{s \in (t-w, t]} x_s \ge m$ 记为"簇发状态"；count = 进入簇发的次数，seconds = 处于簇发的累计秒数——quote stuffing（报价轰炸）视角。
  • ccp__flick：桶内 1 秒中间价的标准差 $\mathrm{std}(m_t)$——前三个量"变动多不多"，这个量"晃动大不大"。

类 5｜事件响应速度

模板：盘口事件 → 开短窗 → 数跟随行为

  • ccp__sresp：每次报价改善（买一上抬或卖一下压）后开一个响应窗口，找到第一个响应事件（同侧追单 / 同侧撤单 / 对改善价的成交）计 1，全日累计——市场对新报价的整体反应速度。
  • kkst__newlevel_maker_lift：L1 发生不利变动后，$\mathrm{mean}\Big(\dfrac{\text{窗口内新报单中被标记订单的占比}}{\text{全日被标记订单占比}}\Big)$——大于 1 说明被标记群体在盘口变化后抢先重新报价。

类 6｜主动成交 / 知情性

核心：taker 行为与价格变动的先后关系

  • ccp__ioc：主动吃单订单计数 $\#\{i : \text{is\_taker}_i\}$。
  • kkst__aggressive_ratio_flagged：$\dfrac{\sum_{\text{主动方被标记的成交}} q}{\sum_{\text{全部主动成交}} q}$——被标记群体吃掉了多大比例的主动成交量。
  • kkst__snipe_taker_lift：每次不利价格变动之前取最近 $k$ 笔成交，算可疑 taker 的量占比，再除以全日基准占比后取平均——大于 1 = 价格要动之前，可疑资金先动手（狙击旧报价）。
  • kkst__flow_price_beta_lag0~lag5：令 $f_t$ 为第 $t$ 秒被标记订单的带符号成交量（买 $+$ 卖 $-$），$\beta_\ell = \dfrac{\mathrm{Cov}(f_t,\ \Delta m_{t+\ell})}{\mathrm{Var}(f_t)}$，$\ell = 0, \dots, 5$——被标记资金流对未来中间价变动的预测系数。
  • kkst__flow_price_beta0：备注：代码级复制 = beta_lag0，纯冗余。

类 7｜成交经济后果

核心：成交价相对当前/未来中间价的偏离；HJM 验证器

对每笔成交 $i$（价格 $p_i$，成交时中间价 $m_0$，$\Delta$ 秒后中间价 $m_1$，方向 $d_i = +1$ 买方主动 / $-1$ 卖方主动）：

  • hjm__espread_bps：有效价差 $ES_i = d_i\,\dfrac{p_i - m_0}{m_0}$——成交那一刻付出的即时成本。
  • hjm__rspread_bps：已实现价差 $RS_i = d_i\,\dfrac{p_i - m_1}{m_0}$——扣掉后续价格漂移后，做市方真正赚到的部分。
  • hjm__advsel_bps：逆向选择成本 $AS_i = ES_i - RS_i$。备注：恒等式差值，三者只有两个自由度。面板值均为成交量加权平均 × $10^4$（bps）。
  • hjm__trade_size：平均单笔成交股数——算法拆单越狠，这个数越小。

类 8｜撤单–成交交互（fade / spoof 形态）

每笔成交（时刻 $t$、价格 $p$、量 $q$）后开 $w$ 秒窗口，统计 maker 同侧、同价（或 ± 价格带）的撤单总量 $C$：

  • kx__fade_partial_prob：$\Pr(C > 0)$——成交后同侧同价"缩手"的概率。
  • kx__fade_full_prob：$\Pr(C \ge q)$——撤掉的比成交的还多。备注：嵌套关系，恒有 partial ≥ full。

备注：与类 1 的区别在于这里看撤单与成交事件的时间排布，是本项目唯一直接刻画 spoofing 形态的指标，逻辑独立性较高。

类 9｜弱监督模型综合（GB）

用 11 个订单特征（寿命、离盘口档数、相对报单量、成交比例、是否主动、终态、时段……）训练梯度提升模型：正种子 = EE step1 ∪ SRuns 命中订单，负种子 = 长寿 + 正常完结 + 不超过中位数量的"类人"订单，其余为灰区。模型输出每笔订单的"疑似算法概率" $\hat p_i$。

  • gb__sum_p：$\sum_i \hat p_i$——全日"算法嫌疑总量"（按笔）。
  • gb__sum_p_amount：$\sum_i \hat p_i \cdot (q_i \times p_i)$——按订单金额加权的嫌疑总量。
  • gb__seed_pos / gb__seed_neg：正/负种子订单数。备注：训练簿记量；由于训练标签来自类 1+2，GB 分数天然与那些类共线，是"非线性重组"而非独立信号源。
