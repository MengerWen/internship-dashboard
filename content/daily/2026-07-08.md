# SRUNS 因子重构：从订单数阈值改到消息数口径

## 1 问题诊断

旧版 `ccp__sruns` 有两个核心错位：

1. 链长阈值卡在 `n_orders >= 10`，而 H&S / CCP 口径和 lowlat 统计口径都是 `message_count >= 10`。一条撤挂链通常包含提交、撤单、成交等多类消息，所以旧阈值实质上过严。
2. 链接规则只要求同方向、限价、数量精确相等；在 1000ms 窗口内不限制重挂价差、不检查通道。数量条件过严，价格条件又过松。

本轮改动把主列名 `ccp__sruns` 保留下来，但语义改为 `message_count >= run_min_len_msg`。同时新增三个家族列：`ccp__sruns_all`、`ccp__sruns_msg10`、`ccp__sruns_wt`。

## 2 代码落地

`lowlat.runs.build_runs` 新增三类可选约束：

- `abs_qty_tol_lots`：绝对数量容差，按 1 手 = 100 股换算；
- `price_band_ticks`：重挂价格相对原订单价格的最大 tick 距离；
- `require_same_channel`：要求重挂单与原撤单同通道。

默认值保持旧行为，因此旧 notebook 和历史脚本不会因为接口变化立刻失效。`links` 输出新增 `d_qty`、`d_price_ticks`、`same_channel` 三个证据列，供校准图直接使用。

`algobench.m3_ccp` 改为一次 `build_runs` 后派生四个 SRUNS 变体：

| 列名 | 含义 |
| --- | --- |
| `ccp__sruns_all` | 全部成链 run 计数 |
| `ccp__sruns` | 主口径，`message_count >= run_min_len_msg` |
| `ccp__sruns_msg10` | 论文可比口径，`message_count >= 10` |
| `ccp__sruns_wt` | 链接强度，$\sum(n\_orders-1)$ |

## 3 当前冻结参数

当前配置写入：

| 参数 | 值 |
| --- | ---: |
| `sruns_link_window_ms` | 1000 |
| `sruns_qty_tol` | 0.1 |
| `sruns_abs_qty_tol_lots` | 1 |
| `sruns_price_band_ticks` | 10 |
| `sruns_require_same_channel` | false |
| `run_min_len_msg` | 6 |

这些参数已经进入 `configs/default.yaml`。校准脚本已按计划 full 模式跑完 6 股票 × 3 日期样本，覆盖高/中/低流动性各 2 只。

## 5 已知限制

full 样本暴露了一个需要继续处理的诊断点：当前“时间平移安慰剂”产生的链接率高于真实流，说明这个 placebo 构造还不能直接解释为噪声上界。参数冻结主要依据覆盖率、链长生存曲线、稳定性平台和规则可解释性；A 图中的 excess 曲线保留为诊断，不单独作为选参证据。

本轮未修改 `m6_gb.positive_run_min_len`、KKST、PCA、HJM；这些仍按计划登记为后续项。
